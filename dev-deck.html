Security: Phase 1 Implementation

✓ OAuth 2.0 for all integrations (Meta, Google, QB, Plaid)
Nicole logs in directly to each service — not to you
✓ API keys never in frontend
All calls proxied through backend (Anthropic, etc.)
✓ localStorage for data storage
All data stays on Nicole's device, not on your servers
✓ HTTPS only (GitHub Pages auto-enforces)

Security: Phase 2 Enhancements

✓ Database encryption at rest
Sensitive fields (account numbers, SSNs) encrypted in PostgreSQL
✓ Encrypted token storage
API tokens stored encrypted in backend, never exposed
✓ Authentication layer
Email + password or OAuth (Google/Microsoft)
✓ Audit logs
Full record of who accessed/changed what and when
✓ Quarterly key rotation
Automatic API key refresh schedule

Security: OAuth Best Practices

  • State parameter: Always include to prevent CSRF attacks
  • Scope minimization: Only request permissions actually needed
  • Token expiration: Implement refresh token logic
  • Redirect URI whitelisting: Only allow registered URLs
  • Token rotation: Refresh monthly (calendar reminder)

Security: Data Privacy

Phase 1

  • All data stays on Nicole's device (localStorage)
  • No sync between devices
  • No backup on your servers
  • Nicole's data at risk if device is lost

Phase 2

  • Database encryption + TLS in transit
  • Automated backups
  • Access audit logs for compliance

FAQ

Q: How do I test locally?
A: python3 -m http.server 8000

Q: Custom domain (meridian.dnaagency.com)?
A: Repo Settings → Pages → Custom domain. Add CNAME record to registrar.

Q: localStorage quota exceeded?
A: Safari ~50MB, Chrome ~10MB. If 500+ contacts, implement Phase 2.

Q: Multi-device in Phase 1?
A: Not recommended. Use Phase 2 for multi-device sync.
